# PakkaQuote Website — Multilingual Support Design

**Date:** 2026-04-05
**Status:** Approved
**Languages:** English (en), Tamil (ta), Hindi (hi), Marathi (mr), Gujarati (gu)

---

## Overview

Add multilingual support to the PakkaQuote static marketing website (4 pages: index, features, about, faq). English source files remain the editable source of truth. A Node.js build script extracts visible text, translates via Google Cloud Translation API, and generates pre-built static HTML files in language-specific subdirectories.

## Architecture: Extract-Translate-Inject

### Build Script (`scripts/translate.js`)

Single Node.js script with three phases:

**Phase 1 — Extract:**
- Parse each HTML file using `cheerio` (DOM parser)
- Walk all visible text nodes and extract translatable strings
- Skip: `<script>`, `<style>`, `<code>` tags
- DO translate: `placeholder`, `alt`, `aria-label`, `title` attributes
- Skip brand terms listed in `i18n/no-translate.json` (e.g., "PakkaQuote", "WhatsApp")

**Phase 2 — Translate:**
- Send extracted strings to Google Cloud Translation API in batches
- Cache translations in `i18n/cache/<lang>.json`
- On re-runs, only translate new/changed strings (cache hit = skip API call)
- API key read from `.env` as `GOOGLE_TRANSLATE_API_KEY`

**Phase 3 — Inject:**
- Clone each HTML file per target language
- Replace English text with cached translations
- Update `<html lang="...">` attribute
- Add `hreflang` link tags to `<head>`
- Inject language switcher component into navbar
- Rewrite internal nav links to stay within language (e.g., `/ta/features.html`)
- Rewrite asset paths (CSS, JS, images) to use absolute paths (`/css/styles.css`, `/js/main.js`) so they resolve correctly from subdirectories
- Write output to `<lang>/<page>.html`

### npm Scripts

- `npm run translate` — full extract → translate → inject for all languages
- `npm run translate -- --lang ta` — single language only

### Dependencies Added

- `cheerio` — HTML parsing and manipulation
- `@google-cloud/translate` — Google Cloud Translation API v2

## File Structure

```
/                            <- English (source, unchanged)
├── index.html
├── features.html
├── about.html
├── faq.html
├── ta/                      <- Tamil (generated, gitignored)
│   ├── index.html
│   ├── features.html
│   ├── about.html
│   └── faq.html
├── hi/                      <- Hindi (generated, gitignored)
│   └── ...
├── mr/                      <- Marathi (generated, gitignored)
│   └── ...
├── gu/                      <- Gujarati (generated, gitignored)
│   └── ...
├── i18n/
│   ├── cache/               <- Translation cache (committed to git)
│   │   ├── ta.json
│   │   ├── hi.json
│   │   ├── mr.json
│   │   └── gu.json
│   └── no-translate.json    <- Brand terms to skip
├── scripts/
│   └── translate.js
```

**Generated language folders** (`ta/`, `hi/`, `mr/`, `gu/`) are **gitignored** — they are regenerated on deploy.

**Translation cache** (`i18n/cache/`) is **committed** — ensures re-runs are fast and only new strings hit the API.

## Language Switcher UI

- Dropdown in the navbar, right side, before the "Login" button
- Shows current language in its native script:
  - English, தமிழ், हिन्दी, मराठी, ગુજરાતી
- On selection, navigates to the same page in the chosen language
- Current language is highlighted/disabled in the dropdown
- Pure HTML + CSS + vanilla JS (no framework)
- Added manually to the 4 English source files (one-time change)
- Injected by the build script into generated language pages
- Mobile-friendly within existing responsive nav

**No auto-detection.** Users land on whichever URL they visit. No browser language sniffing or redirects.

## SEO & Metadata

### hreflang Tags

Injected into `<head>` of every page:

```html
<link rel="alternate" hreflang="en" href="https://pakkaquote.com/features.html" />
<link rel="alternate" hreflang="ta" href="https://pakkaquote.com/ta/features.html" />
<link rel="alternate" hreflang="hi" href="https://pakkaquote.com/hi/features.html" />
<link rel="alternate" hreflang="mr" href="https://pakkaquote.com/mr/features.html" />
<link rel="alternate" hreflang="gu" href="https://pakkaquote.com/gu/features.html" />
<link rel="alternate" hreflang="x-default" href="https://pakkaquote.com/features.html" />
```

### Translated Metadata

- `<meta name="description">` — translated
- `og:title` and `og:description` — translated
- Structured data (JSON-LD) `name` and `description` fields — translated

### Not Translated

- `og:image` — single English image shared across all languages
- Brand name "PakkaQuote" — kept in English everywhere
- URL slugs — no transliterated paths

### Sitemap

`sitemap.xml` is regenerated by the build script to include all language variants with `xhtml:link` alternates per URL.

## What Stays Unchanged

- **CSS** — shared `css/styles.css`. Indian scripts (Devanagari, Tamil, Gujarati) render via browser system font fallback.
- **JavaScript** — `js/main.js` and `js/tracking.js` unchanged. FAQ accordion, form submission, analytics all work as-is.
- **HubSpot form** — submits in whatever language the user types. No backend changes.
- **Analytics** — GTM/GA4 tracking unchanged. A `language` property is added to the dataLayer for GA4 segmentation.
- **Assets** — images, favicon, OG image all shared.
- **Cookie consent banner** — text translated along with the rest of the page.

## Configuration

### `.env`

```
GOOGLE_TRANSLATE_API_KEY=your-api-key-here
```

### `i18n/no-translate.json`

```json
{
  "terms": [
    "PakkaQuote",
    "WhatsApp",
    "PDF",
    "GTM",
    "GA4",
    "HubSpot",
    "API"
  ]
}
```

## Workflow

**To update content:**
1. Edit the English HTML files as usual
2. Run `npm run translate`
3. Deploy (all language folders are regenerated)

**To add a new language:**
1. Add the language code to the config in `scripts/translate.js`
2. Run `npm run translate`
